## 一、实验内容

#### 1.选题说明
本次实验选择 arXiv 中的 `cs.CL`方向作为数据采集对象。
选择这一方向主要有以下两个原因。

首先，自然语言处理是当前人工智能领域中非常活跃的研究方向之一。尤其是在大语言模型快速发展的背景下，文本生成、问答系统、机器翻译、信息抽取等任务都取得了明显进展，因此这一领域具有较强的研究价值和现实意义。

其次，我在课外学习中接触并实践过开源项目 MiniMind。通过阅读和运行该项目，我对语言模型的基本原理，如文本表示、序列建模和生成机制，有了更直观的理解。同时，我也逐渐认识到高质量数据在自然语言处理任务中的重要性。因此，本次选择 `cs.CL` 方向，一方面与我的兴趣相符，另一方面也可以把已有的开源实践经验与真实科研论文数据结合起来，加深对该领域研究内容的理解。

综上，选择 `cs.CL` 作为本次实验的抓取对象，既符合当前人工智能技术的发展热点，也与我个人的学习经历和兴趣方向较为一致。

#### 2.爬虫介绍
本次实验实现的是一个面向 arXiv 的论文爬虫程序，主要功能包括论文数据抓取、PDF 下载以及 PDF 文本提取三个部分。整个项目按照功能拆分为 `crawler.py`、`download_pdf.py`、`pdf_to_txt.py` 和 `main.py` 四个脚本。

其中，`crawler.py` 负责调用 arXiv API 获取论文信息，并按照给定的分类、时间范围和数量要求抓取数据。抓取到的每篇论文信息会保存为一条 JSON 记录，主要字段包括论文编号、标题、作者、摘要、发布日期、分类、论文链接。所有结果最终统一保存到 `papers.json` 文件中。

在完成数据抓取后，`download_pdf.py` 会读取 `papers.json`，根据每篇论文的 `arxiv_id` 拼接出对应的 PDF 下载链接，并将论文保存到 `paper_pdf` 文件夹中。为了避免重复下载，程序会先判断目标文件是否已经存在，如果存在则直接跳过。

随后，`pdf_to_txt.py` 会继续读取 `papers.json` 中记录的 PDF 路径，使用 `pdfminer.six` 将 PDF 文件提取为 TXT 文本，并保存到 `paper_txt` 文件夹中。如果某篇论文在提取过程中出现异常，程序只会输出错误信息，不会中断整个流程。

最后，`main.py` 用于串联上述三个步骤，使整个实验流程可以通过一条命令完成。整体流程较为清晰：先抓取数据，再下载 PDF，最后提取文本。

#### 3.实现过程中遇到的问题及解决方法
在实现本次爬虫的过程中，我主要遇到了以下几个问题。

（1）arXiv API 返回的数据需要分页获取。由于一次请求返回的论文数量有限，如果只请求一页数据，无法满足作业中“抓取不少于 200 篇论文”的要求。为了解决这个问题，我在程序中设置了分页参数，通过循环更新 `start` 和 `max_results`，分批请求论文数据，直到达到设定数量为止。

（2）抓取结果中可能会出现不符合日期要求的论文，或者不同批次之间存在重复记录。为了解决这个问题，我在程序中加入了日期过滤逻辑，并使用论文的 `arxiv_id` 作为唯一标识进行去重，这样可以保证最终保存的数据既满足时间范围要求，也不会重复。

（3）在重复运行程序时，如果已经下载过部分 PDF 或已经生成过部分 TXT，再次全部处理会浪费时间。为了解决这个问题，我在下载和转换前都增加了文件存在性判断，若目标文件已经存在，则直接跳过。这样不仅提高了效率，也方便后续多次调试程序。
